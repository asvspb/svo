# Правила вайбкодинга (Coding Guidelines)

Цель: единый инженерный вайб команды — читаемый, предсказуемый, поддерживаемый код. Снижаем стоимость сопровождения, ускоряем онбординг и ревью.

Содержание:
- [Style Guide](./style_guide.md)
- [Testing Guide](./testing_guide.md)
- [CI/CD Guide](./cicd_guide.md)

## Общие принципы
- Предпочитайте простоту: очевидные решения важнее сложных хаков.
- Малые, атомарные изменения: один PR — одна логическая задача.
- Самодокументируемый код: говорящие имена > лишние комментарии.
- Архитектурная целостность важнее локальных оптимизаций.
- Пишите для «ревьюера через полгода».

## Проектная специфика (кратко)
- Гео-данные: CRS обязательно; для площадей — равновеликие проекции; валидируйте геометрию и входные схемы; используйте overlay/difference с документированными допущениями.
- Playwright: параметризуйте headless, избегайте `sleep`; ждите конкретные ответы/селекторы; фильтруйте перехват сети; логируйте и ретрайте.
- Telegram-бот: разделяйте хендлеры и бизнес-логику; FSM для сценариев; rate limiting; токены и админ-ID — только через ENV.

Подробности и конкретные практики — в разделах Style, Testing и CI/CD.
